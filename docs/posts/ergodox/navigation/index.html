April 27, 2016 Ergodox Navigation Layer

When I decided on the ergodox, it was mostly because I wanted a split keyboard that didn't have staggered rows. The fact that it was entirely programmable was a neat perk. Before the ergodox came, I spent a lot of time thinking about how I wanted to lay out the keys. I've done a lot of things that make sense to me, which I haven't seen in any other layouts that I've come across.

In particular, I really like Vim keybindings, and wanted to see how close I could get to a vim experience just by adding a navigation layer. It turned out really well. I can very quickly navigate and select text, and though there are still a few Vim keybinds that I miss, for the most part I haven't looked back. Here is a look at my main layers.

Layer 0 - Colemak + Backspace

I currently type on a colemak layout, which places ; to on QWERTY 'p'. Since I have an entire layer dedicated to symbols, I decided that I can get by without ; in the main layout. Also, as a programmer having to reach up for ; is a bit annoying with colemak, so I find it a lot nicer to be in its usual right pinky position, just on a symbol layer. I chose to move 'z' to ';', and replaced 'z' with a backspace. Backspace is now in the main layout, and it's magical!

Layer 1 - Symbol layer

I have a symbol layer activated by one of the thumb buttons. It has a lot of () {} [] in very friendly positions so it's quite comfortable to program on. This is pretty standard for an ergodox layout. I also have very similar layer for numbers.

Layer 2 - Navigation Layer

While not a Vim user often, I am a very big fan of vim keybinds. Depending on the IDE I'm using at the time (IntelliJ currently) I have better or worse key bindings. Instead, I've tried to recreate what I can of a vim feel, by giving myself a navigation layer in my ergodox layout.

I do really miss some of the power of vim, but all in all, it makes navigation and editing reasonably nice accross all applications.

Holding the right thumb accesses the navigation layer. If I'm currently in the symbol layer (hollding left thumb), then right thumb pushes the layer rather than toggling it. So, both thumbs will lock the navigation layer

This gives me arrows under my right hand, as well as Home and End. Right hand home row becomes shift, ctrl, alt, so that when I use the left hand arrows I can be holding shift or control shift to quickly navigate by words and select by words. I also have home and end, so selecting text to the beginning or the end of the line is very easy as well.

I've recently added a number of hotkeys for dealing with windows. I'm using VirtuaWin at the moment. I use ctrl+alt+arrows to move about desktops. That's a lot of buttons, but they're all right under my fingers in home row in the navigation layer!

The lower letter row has two keys so far

  • #! (windows key + alt) This plus an arrow is used to move windows to other desktops.
  • ^#W (windows key + ctrl + W) This pops up VirtuaWin's window selector

My upper letter row has two hotkeys: maximize window and move window to other monitor.

A few other cool keybindings that have made their way their way onto this layer:

  • Bottom row of the left will become cut, copy, and paste, in the usual x,c,v key positions.
  • !W (alt + w) switches between panels in Sublime Text
  • ^W (ctrl + w) grows the current selection in Sublime Text
  • ^Z (undo) is on the right hand.
  • +F11 and ^F11 (shift + F11 and ctrl + F11) These are bound to set and goto bookmarks in sublime and intellij. This gets me close to the behaviour of m (mark) and ' (go to mark) in vim.
